# Security Policy

TPT-AV is security software, so we take vulnerabilities seriously and appreciate
responsible disclosure.

## Supported versions

As an early-stage project, security fixes are applied to the latest released
version and the `main` branch. Please upgrade to the latest release before
reporting an issue.

## Reporting a vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

Instead, use one of the following private channels:

- Open a [GitHub Security Advisory](https://github.com/tpt-av/tpt-av/security/advisories/new)
  (preferred), or
- Email the maintainers at **security@tpt-av.dev**.

Please include:

- A description of the vulnerability and its impact.
- Steps to reproduce or a proof of concept.
- Affected version(s) and platform(s).
- Any suggested remediation, if known.

## What to expect

- We aim to acknowledge reports within **72 hours**.
- We will investigate, keep you informed of progress, and coordinate a fix and
  disclosure timeline with you.
- With your permission, we will credit you in the release notes.

Thank you for helping keep TPT-AV and its users safe.
